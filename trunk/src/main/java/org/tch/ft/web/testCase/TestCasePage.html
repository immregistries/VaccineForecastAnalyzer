Test Case

A Test Case represents a specific scenario and contains enough information for a basic patient record to be created in an IIS.

The Test Case detail page shows the basic information for the test case, the associated events (vaccinations) and the status of this test case within the context of the currently selected Test Panel. If the test case is part of the current test panel and you are assigned Expert or Admin role to the Task Group, you can edit the test case.

Test Case

  • Label A short description of what this test case tests.
  • Description A complete description of what this test case tests.
  • Patient The assigned (not-real) name and sex of the patient.
  • DOB The date of birth for testing.
  • Evaluation Date The date to target the forecast at.

Events

  • Type Type of event that occurred. This is normally a Vaccination, but could also be a contraindicatin or other condition.
  • Item The particular concept, normally a specific vaccination.
  • CVX The CDC assigned CVX code, if this is a vaccination.
  • MVX The CDC assigned manufacturer code, if this is a vaccination.
  • Date The date the event occurred, normally the vaccination date.

Assigned to Test Panel

A single test case can be assigned to more than one panel. This section shows the values for this test case for the currently selected test panel. This means that a single test may "Pass" under one test panel and "Fail" in another. This situation would occur when a single test case works for one IIS and not for another.

  • Category Name The category this test case is assigned under in the test panel.
  • Test Case Number The unique number assigned by the Task Group when this test case was assigned to the test panel.
  • Include Status Whether nor this test case is assigned as part of this test panel. Once added a test case is never fully removed, instead it is marked as "Excluded". "Proposed" test cases are ones that have been provisionally added and are waiting for approval from the rest of the group.
  • Result Status Indicates the current status of this test case for testing purposes. This is used to help communication between Task Group experts, and is not automatically updated when a test is run.

Test Results

A comparison of the actual vs expected values. This page will show all expected and actual values with the currently selected Task Groups expected values compared with the currently selected Software. For most test cases there is only one set of actuals vs expected, but some test cases, particular for combinations, may have additional tests for each antigen. In these case the actuals and expected are repeated.

Set Expected Values for ...

This form allows for quickly indicated the expected values for this test case. Every test case can have an official expected value from any Task Group that has this test case in a Test Panel. In addition, any Expert can add their personal expected values. If you are currently assigned as an expert on a panel that includes this test case then your expectations will become the official expectations for this test panel. Otherwise your expectations will be displayed only under your name. (You do not have the ability to change the expectations for any Task Group that you are not an Expert or Admin on. )

  • Dose The dose number that is expected next.
  • Valid The earliest date when the next (or first) vaccination could be given. (This does not include any additional grace period that the forecaster may allow for in retrospective analysis.)
  • Due The date when this vaccination would normally be recommended. This includes any acceleration due to being behind schedule. This is the date when the patient would normally be asked to return for the first or additional vaccinations.
  • Overdue The first date when the next (or first) vaccination should have already been given. Optimally the patient should be scheduled for and received a vaccination by this date. After this point the patient is considered "behind". This value has a higher variability among systems as it depends on both recommendations and local policy.

Change Test Status for ...

This table allows Task Group experts to quickly change the status of this test case for the currently selected test case panel in context of how well this forecast test case is handled by the primary software system listed in the title.

Comments

Any Expert may leave comments on any test case and read all comments on a test case. Remember, do not put confidential or private information in the comments as they will be visible to any registered system user. The comments are a common bulletin board for notes and comments that can be shared between Task Group members and other Experts.
